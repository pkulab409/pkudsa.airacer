## 赛车模型参数

### 通用参数
所有赛车模型共享以下通用参数：

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| name | SFString | "car_xxx" | 赛车名称 |
| translation | SFVec3f | 0 0 0.302 | 赛车位置 (x, y, z) |
| rotation | SFRotation | 0 0 1 0 | 赛车旋转 (x, y, z, angle) |
| controller | SFString | "car" | 控制器名称 |
| customData | SFString | "" | 自定义数据 |
| bodySize | SFVec3f | 4.2 1.8 0.12 | 车身尺寸 (长, 宽, 高) |
| mass | SFFloat | 800.0 | 车辆质量 (kg) |

## 赛道元素参数

### 道路系统

#### Road（直道）
```proto
Road {
  translation x y z           # 道路中心位置
  rotation 0 0 1 angle        # 道路旋转角度 (弧度)
  name "road_xxx"             # 道路名称
  width 10                    # 道路宽度
  rightBarrier TRUE           # 右侧是否有护栏
  leftBarrier TRUE            # 左侧是否有护栏
  bottom TRUE                 # 底部是否有地面
  wayPoints [                 # 路径点列表
    x1 y1 z1
    x2 y2 z2
    ...
  ]
  splineSubdivision 8         # 曲线细分度 (仅曲线道路)
  appearance {                # 外观设置
    baseColor r g b           # 颜色 (0-1)
    roughness 0.9             # 粗糙度
    metalness 0               # 金属度
  }
}
```

**参数说明**：
- `translation`: 道路中心点的世界坐标
- `rotation`: 道路旋转角度，0表示沿X轴方向
- `width`: 道路宽度，默认10单位
- `wayPoints`: 定义道路路径的关键点
- `splineSubdivision`: 曲线平滑度，值越大曲线越平滑

#### CurvedRoadSegment（曲线段）
```proto
CurvedRoadSegment {
  translation x y z           # 曲线段中心位置
  rotation 0 0 1 angle        # 曲线段旋转角度
  name "curved_seg_x"         # 曲线段名称
  width 10                    # 道路宽度
  rightBarrier TRUE           # 右侧护栏
  leftBarrier TRUE            # 左侧护栏
  bottom TRUE                 # 底部地面
  curvatureRadius 10.5        # 曲率半径 (单位)
  appearance {                # 外观设置
    baseColor r g b           # 颜色
    roughness 0.9             # 粗糙度
    metalness 0               # 金属度
  }
}
```

**参数说明**：
- `curvatureRadius`: 曲率半径，值越小曲线越急
- 其他参数与Road类似

**赛道布局**：
1. **road_south_straight**: 南侧直道
   - 位置: (-20, 125, 0.1)
   - 旋转: 90度 (沿X轴)
   - 路径: (-125, 0, 0) → (125, 0, 0)

2. **road_ne_curve**: 东北曲线
   - 位置: (42, 259, 0.1)
   - 路径: (-55, 0, 0) → (0, 30, 0) → (55, 0, 0)

3. **road_north_straight**: 北侧直道
   - 位置: (100, 125, 0.1)
   - 旋转: -90度 (沿X轴)
   - 路径: (-125, 0, 0) → (125, 0, 0)

4. **road_sw_curve**: 西南曲线
   - 位置: (41, -10, 0.1)
   - 路径: (-55, 0, 0) → (0, -30, 0) → (55, 0, 0)

5. **curved_seg1**: 曲线段1
   - 位置: (-9.3, -1, 0.1)
   - 旋转: 165度
   - 曲率半径: 10.5

6. **curved_seg2**: 曲线段2
   - 位置: (90, -1, 0.1)
   - 旋转: -75度
   - 曲率半径: 10.5

7. **curved_seg3**: 曲线段3
   - 位置: (-10, 249, 0.1)

### 检查点系统

#### Checkpoint（检查点）
```proto
Solid {
  translation x y z           # 检查点位置
  rotation 0 0 1 angle        # 检查点旋转角度
  name "checkpoint_x"         # 检查点名称
  children [
    Shape {
      appearance PBRAppearance {
        baseColor 0 0.8 1     # 颜色 (青蓝色)
        transparency 0.6      # 透明度 (0-1)
        roughness 0.3         # 粗糙度
        metalness 0           # 金属度
      }
      geometry Box {
        size 10 0.1 2         # 尺寸 (长, 宽, 高)
      }
    }
  ]
}
```

**参数说明**：
- `baseColor`: 检查点颜色，青蓝色 (0, 0.8, 1)
- `transparency`: 透明度，0.6表示半透明
- `size`: 检查点尺寸，长10单位，高2单位

**检查点位置**：
1. **checkpoint_1**: 起点/终点检查点
   - 位置: (-20, 125, 0.1005)
   - 旋转: -90度

2. **checkpoint_2**: 东北角检查点
   - 位置: (40, 279, 0.1005)
   - 旋转: 97.5度

3. **checkpoint_3**: 北侧检查点
   - 位置: (100, 125, 0.1005)
   - 旋转: 0度

4. **checkpoint_4**: 西南角检查点
   - 位置: (40, -30, 0.1005)
   - 旋转: 82.5度

**检查点功能**：
- 赛车必须按顺序通过所有检查点
- 每通过一个检查点记录一次进度
- 完成所有检查点通过算作完成一圈
- 用于判定赛车是否偏离赛道或逆行

### 起跑线

#### Start/Finish Line（起跑线）
```proto
Solid {
  translation x y z           # 起跑线位置
  rotation 0 0 1 angle        # 起跑线旋转角度
  name "start_finish_line"    # 起跑线名称
  children [
    Shape {
      appearance PBRAppearance {
        baseColor 0.05 0.05 0.05  # 颜色 (深灰色)
        roughness 0.9             # 粗糙度
        metalness 0               # 金属度
      }
      geometry Box {
        size 0.5 10 0.001         # 尺寸 (长, 宽, 高)
      }
    }
  ]
}
```

**参数说明**：
- 位置: (-20, 125, 0.1005)
- 旋转: -90度
- 尺寸: 长0.5单位，宽10单位，高0.001单位（极薄）

**功能**：
- 赛车出发和完成比赛的标记线
- 用于记录赛车完成圈数
- 作为比赛成绩统计的基准

### 发车信号灯

#### GenericTrafficLight（交通信号灯）
```proto
GenericTrafficLight {
  translation x y z           # 信号灯位置
  rotation 0 0 1 angle        # 信号灯旋转角度
  state "green"               # 当前状态: "green" | "yellow" | "red"
}
```

**参数说明**：
- `state`: 信号灯状态
  - "green": 绿灯（允许出发）
  - "yellow": 黄灯（准备出发）
  - "red": 红灯（禁止出发）

**信号灯位置**：
- **发车信号灯**: (-13.5, 130, 0.1)
- 旋转: -90度
- 当前状态: "green"

**功能**：
- 控制赛车出发时机
- 绿灯亮起时赛车可以出发
- 黄灯闪烁时提醒赛车准备
- 红灯时赛车必须停止

```proto

      geometry Box {

      }
    }
  ]
}
```

**参数说明**：
- 颜色: 浅灰色 (0.96, 0.96, 0.96)
- 尺寸: 长4单位，宽2单位，高0.001单位

**预备车位布局**：
1. **pre_grid_1**: (-23, 123.7, 0.1005)
2. **pre_grid_2**: (-17, 123.7, 0.1005)
3. **pre_grid_3**: (-17, 116, 0.1005)
4. **pre_grid_4**: (-23, 116, 0.1005)
5. **pre_grid_5**: (-17, 108, 0.1005)
6. **pre_grid_6**: (-23, 108, 0.1005)

**功能**：
- 赛车出发前的停放位置
- 每个车位可停放一辆赛车
- 赛车在发车信号灯变绿后从车位出发

## 开发注意事项

1. **坐标系说明**
   - Webots使用右手坐标系
   - X轴: 向右为正
   - Y轴: 向前为正
   - Z轴: 向上为正

2. **单位说明**
   - 所有距离单位为米
   - 所有角度单位为弧度
   - 时间单位为秒

3. **性能优化**
   - 合理设置仿真步长
   - 优化传感器采样频率
   - 减少不必要的计算

4. **调试技巧**
   - 使用Webots内置调试工具
   - 查看传感器数据可视化
   - 记录赛车轨迹和传感器数据

5. **测试建议**
   - 先测试单个功能模块
   - 逐步集成测试
   - 进行多轮仿真测试
   - 记录测试结果和问题
